# 환경변수 설정 가이드

로컬 개발 서버를 실행하기 전에 아래 단계를 순서대로 완료하세요.

---

## 1. `.env.local` 파일 생성

```bash
cp .env.example .env.local
```

이후 아래 섹션을 참고하여 각 값을 채워넣습니다.

---

## 2. API 키 발급 안내

### Anthropic API Key (`ANTHROPIC_API_KEY`)

Claude 모델(claude-sonnet-4-6) 호출에 사용됩니다.

1. https://console.anthropic.com 에서 로그인 또는 회원가입
2. 좌측 메뉴 **API Keys** → **Create Key**
3. 생성된 키를 복사하여 `.env.local`에 입력

```
ANTHROPIC_API_KEY=sk-ant-api03-...
```

---

### Firebase (서버용 — Service Account)

`FIREBASE_PROJECT_ID`, `FIREBASE_PRIVATE_KEY`, `FIREBASE_CLIENT_EMAIL`

Express API 서버가 Firestore에 접근할 때 사용합니다.

1. https://console.firebase.google.com 에서 프로젝트 생성 또는 선택
2. **프로젝트 설정** (톱니바퀴 아이콘) → **서비스 계정** 탭
3. **새 비공개 키 생성** 버튼 클릭 → JSON 파일 다운로드
4. JSON 파일에서 아래 값을 추출하여 입력

```
FIREBASE_PROJECT_ID=your-project-id
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-xxxxx@your-project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\nMIIE...\n-----END PRIVATE KEY-----\n"
```

> **주의**: `FIREBASE_PRIVATE_KEY`는 줄바꿈이 `\n`으로 이스케이프된 상태 그대로 큰따옴표로 감싸서 입력해야 합니다.

---

### Firebase (클라이언트용 — Web Config)

`NEXT_PUBLIC_FIREBASE_*` 환경변수들

Next.js 프론트엔드에서 Firebase Auth / Firestore에 접근할 때 사용합니다.

1. https://console.firebase.google.com → 동일 프로젝트 선택
2. **프로젝트 설정** → **일반** 탭 → 하단 **내 앱** 섹션
3. 웹 앱(`</>`)이 없으면 **앱 추가** → 웹 선택
4. **Firebase SDK 구성** → **구성** 선택 시 표시되는 값 입력

```
NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSy...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your-project-id
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-project.appspot.com
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789
NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789:web:abcdef
```

---

### Whisk API Key (`WHISK_API_KEY`)

텍스트→이미지 생성 (MST 주입 포함) 기본 엔진으로 사용됩니다.

1. https://labs.google/flow/about 에서 Whisk API 접근 신청
2. 승인 후 발급된 API 키를 입력

```
WHISK_API_KEY=your_whisk_api_key
```

> **참고**: Whisk API는 현재 제한적 접근(waitlist) 상태일 수 있습니다. 접근 불가 시 Nano Banana API를 단독으로 사용하도록 `WHISK_API_KEY`를 공백으로 유지하세요.

---

### Nano Banana API Key (`NANO_BANANA_API_KEY`)

이미지 생성 A/B 게이팅의 B 채널로 사용됩니다.

1. https://www.nanobanana.io 에서 계정 생성
2. 대시보드 → **API Keys** → 키 생성
3. 생성된 키를 입력

```
NANO_BANANA_API_KEY=nb_...
```

---

### Replicate API Key (`REPLICATE_API_KEY`)

CLIP ViT-L/14 모델을 통한 SCC(스타일 일관성 검증) 파이프라인에 사용됩니다.

1. https://replicate.com 에서 로그인 또는 회원가입
2. 우측 상단 프로필 → **Account settings** → **API tokens**
3. **Create token** → 생성된 토큰 입력

```
REPLICATE_API_KEY=r8_...
```

---

### API 서버 설정

```
API_PORT=4000
API_BASE_URL=http://localhost:4000
NEXT_PUBLIC_API_BASE_URL=http://localhost:4000
```

> 기본값 그대로 사용해도 됩니다. 포트 충돌이 있을 경우 `4000`을 다른 값으로 변경하세요.

## 3. Firebase 초기 설정

### Firebase 에뮬레이터 (실 계정 없이 로컬 개발)

Firebase 실 프로젝트를 아직 만들지 않았다면 에뮬레이터로 바로 시작할 수 있습니다.

```bash
# Firebase CLI 설치
npm install -g firebase-tools

# 에뮬레이터 설치
firebase setup:emulators:firestore

# 에뮬레이터 실행
firebase emulators:start --only firestore
```

에뮬레이터가 실행되면 `.env.local`에 아래 줄을 추가하세요:

```
FIRESTORE_EMULATOR_HOST=localhost:8080
```

`FIREBASE_PROJECT_ID`, `FIREBASE_PRIVATE_KEY`, `FIREBASE_CLIENT_EMAIL`이 설정되지 않은 경우
API 서버는 자동으로 에뮬레이터 모드로 동작합니다.

---

### Firestore 데이터베이스 생성 (실 Firebase 사용 시)

1. https://console.firebase.google.com → 프로젝트 선택
2. **Firestore Database** → **데이터베이스 만들기**
3. **테스트 모드**로 시작 (개발용), 이후 `firebase/firestore.rules` 규칙 배포

### Firestore 보안 규칙 및 인덱스 배포

```bash
# Firebase CLI 설치
npm install -g firebase-tools

# 로그인
firebase login

# 프로젝트 연결
firebase use your-project-id

# 규칙 및 인덱스 배포
firebase deploy --only firestore
```

## 4. 로컬 개발 서버 실행

```bash
# 1. 의존성 설치
npm install

# 2. 환경변수 확인 (.env.local 파일 존재 여부)
ls .env.local

# 3. 전체 개발 서버 실행 (web + api 동시)
npm run dev

# 또는 개별 실행
npm run dev:web   # Next.js :3000
npm run dev:api   # Express  :4000
```

접속:
- 프론트엔드: http://localhost:3000
- API 서버: http://localhost:4000

---

## 5. 환경변수 검증

API 서버가 정상 시작되면 아래 엔드포인트로 확인:

```bash
curl http://localhost:4000/health
```

정상 응답: `{"status":"ok"}`

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| `FirebaseError: Failed to parse private key` | `FIREBASE_PRIVATE_KEY` 형식 오류 | 줄바꿈을 `\n`으로 이스케이프하고 큰따옴표로 감싸기 |
| `Error: ANTHROPIC_API_KEY is required` | 키 미설정 | `.env.local` 파일 존재 및 값 확인 |
| `EADDRINUSE :4000` | 포트 충돌 | `API_PORT`를 다른 값으로 변경 |
| Whisk API 401 | 키 만료 또는 미발급 | Whisk 콘솔에서 키 재발급 |
| SCC 검증 실패 반복 | Replicate 크레딧 부족 | https://replicate.com/account/billing 에서 충전 |
